    Observable¶

    API Refrence¶

    observable(init, path)¶

    import { observable } from "/libraries/observable.js"
function observable(init, path) {

}

    Parameters¶

    Name Type Description
    init Object Object to add observation too, recursively wraps objects with a Proxy to ovveride the getter and setter
    path string Optional, if called with this parameter then all values set on this object will be synced to firebase

    observe(state, observer, key)¶

    import { observe } from "/libraries/observable.js"
function observe(state, observer, key) {}

    Parameters¶

    Name Type Description
    state observable Object to observe changes too
    observer (state, fromDatabase, key, undoing) => void function that will be called when a property of state changes
    key string Optional, if provided observer will only be called when the value of this property changes

    Return Value¶

    Function to remove the observer

    observeKeys(state, observer)¶

    import { observeKeys } from "/libraries/observable.js"
export function observeKeys(state, observer) {}

    Parameters¶

    Name Type Description
    state observable Object to observe key changes too
    observer (state, key, fromDatabase, undoing) => void function that will be called when properties are added or removed from state

    Return Value¶

    Function to remove the observer

    get(state, path, last)¶

    import { get } from "/libraries/observable.js"
export function get(state, path, last) {}

    Parameters¶

    Name Type Description
    state object
    path string deep path on state each property is delimetted by . for example thing.colour
    last any Optional, if provided will be used to set the last property if it is not undefined

    Old Docs¶

    State proxy is a wrapper around a javascript object. It integrates with firebase, wrapping around a regular javascript object and tracking when things are set and get from the object. It can also include where in firebase each object is, and automatically sync between the object and firebase. To setup up a regular proxy for subscribing to value changes or for history use you can just call proxy(objName) to make a proxy that wraps a firebase value you first get the object with const value = await database.get(path) and then create a proxy with the path proxy(value, path). There a couple different things you can subscribe to, the first way to subscribe is to any changes on the database with subscribe(obj, callback) the second way is to if a certain key on an object changes(this key canâ€™t be another object; it should be a primitive: subscribe(obj, callback, key) the third way is subscribing to if a key is added or removed to an object with subscribeKeys(obj, callback).
